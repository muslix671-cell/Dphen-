# Index des sessions Holodeck DPhen

Cet index determine quelles sessions peuvent soutenir la memoire recente au demarrage du programme.

Seules les sessions actives pertinentes au sujet doivent etre chargees. Les archives techniques et les versions remplacees servent a la provenance documentaire, jamais a la memoire vecue des residents.

## Sessions actives

- `004_import_premiere_seance_feuille_personnage.md` : reconstruction de reference de la premiere seance sur la feuille de personnage.

## Sessions archivees — historique technique

- `archive/technical/001_initialisation_holodeck.md` : creation de la structure documentaire.
- `archive/technical/002_test_pipeline.md` : test du pipeline Codex et Git.

Ces fichiers restent hors du chargement normal d'une scene.

## Sessions archivees — remplacees

- `archive/superseded/003_retrospective_feuille_personnage.md` : premiere retrospective, remplacee par la session 004 plus complete.

Une session remplacee peut etre consultee pour verifier la provenance, mais son contenu ne doit pas etre charge en double avec sa version de reference.

## Regle d'ajout

Avant de creer une session, utiliser le prochain numero disponible sans renumeroter l'historique. Ajouter ensuite son statut a cet index : active, technique ou remplacee.
